# Projects

Project-specific pages. Time-bound goals, initiatives, and their status.

## What Goes Here

- Active projects and goals
- Project status and progress
- Artifacts and deliverables
- Key decisions and context
- Timelines and milestones
- Related people and dependencies

## File Naming

Use kebab-case for clarity: `project-name.md`

Examples:
- `personal-website-redesign.md`
- `knowledge-base-migration.md`
- `course-on-systems-thinking.md`

## Page Structure

Include:
- **Status** — Active / On Hold / Completed / Cancelled
- **Goal** — Clear statement of what you're trying to achieve
- **Timeline** — Start date, target completion, milestones
- **Progress** — Current status and next steps
- **Related Concepts** — Links to relevant [[concept-page]]s
- **People** — Collaborators or stakeholders ([[person-name]])
- **Artifacts** — Links to deliverables or important files

## Relationships

Link to concepts that inform the project, and people involved.

Example:
```
Related: [[project-management]], [[time-blocking]]
Team: [[john-doe]], [[jane-smith]]
```
