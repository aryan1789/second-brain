## Overview

A comprehensive guide to effective prompting with Claude, covering general principles and task-specific techniques for maximizing output quality and relevance.

## Core Principles

### 1. Be Clear and Specific
- State your task or question explicitly at the beginning
- Provide detailed context to help Claude understand your needs
- Break complex tasks into manageable steps

### 2. Use Examples
- Provide concrete examples of desired output format or style
- Show Claude the tone, structure, or approach you want
- Reference similar work when available

### 3. Encourage Thinking
- Ask Claude to "think step-by-step" for complex problems
- Request explanations of reasoning alongside answers
- Use structured breakdown for systemic problem-solving

### 4. Iterative Refinement
- View prompting as a conversation, not a one-shot request
- Provide specific feedback: "That's close, but adjust X to be more like Y"
- Refine outputs through targeted modifications

### 5. Leverage Knowledge
- Ask for background information and explanations
- Include relevant context so responses are maximally targeted
- Use Claude's broad knowledge base strategically

### 6. Use Role-Playing
- Ask Claude to adopt specific perspectives (e.g., "Act as a CFO")
- Explore multiple viewpoints by switching roles
- Increase nuance by framing as specific personas

## Task-Specific Techniques

### Content Creation
- **Specify audience:** Who is this for? What's their background?
- **Define tone:** Professional, casual, technical, creative?
- **Define structure:** Provide outline or key sections needed

### Document Analysis & Q&A
- Be specific about what you want summarized
- Reference attached documents by name
- Request citations for answers

### Data Analysis
- Specify desired output format (table, narrative, visual recommendations)
- Ask for specific metrics and key findings
- Request actionable recommendations

### Brainstorming
- Ask for specific number of ideas
- Categorize by use case or impact
- Request in structured format (lists, tables)

## Troubleshooting & Performance

### Reduce Hallucinations
- Allow Claude to acknowledge uncertainty ("It's okay to say you don't know")
- Provide all contextual information upfront
- Break large tasks into smaller sequential steps

### Maximize Quality
- Include all necessary context in each request (Claude doesn't retain between conversations)
- Be explicit about what constitutes a good answer
- Provide constraints and guidelines

## Key Distinctions

- **Bad prompt:** Vague, assumes context, minimal guidance
- **Good prompt:** Specific goal, detailed context, clear success criteria, structured output request
