# Query the Wiki

Your task: Answer a question by synthesizing knowledge from wiki pages.

## Process

1. **Parse the question** — Understand what's being asked. Break it into key concepts if complex.

2. **Find relevant pages** — Read wiki/index.md, then identify and read the wiki pages most relevant to the question. Start with pages directly addressing the topic, then expand to related concepts.

3. **Synthesize an answer** — Draw on the wiki pages to build a response:
   - Ground every claim in a wiki page (cite by page name in the format: "Per [[page-name]], ...")
   - If multiple wiki pages address the question, integrate them and show how they connect
   - Flag contradictions explicitly: "[[page-A]] argues X, but [[page-B]] argues Y. The difference is..."
   - If the wiki has incomplete information, note what's missing

4. **Propose updates (do not write)** — If synthesizing the answer reveals a new insight or connection that should be captured, propose a wiki update:
   - Suggest a new page or an update to an existing page
   - Explain why it's useful
   - Do NOT write the page — ask for permission first

## Guidelines

- Be direct and specific — answer the actual question
- Cite wiki pages, not external sources
- If the wiki doesn't have enough information, say so clearly
- Use the same style as wiki pages: clear, concise, bullet-pointed where appropriate
- Use DD-MM-YYYY for any dates

## When Done

Report:
- Your answer (grounded in wiki pages)
- Pages cited
- Any proposed updates (with reasoning)
